Except: some of the EVTYPE are mangled and typo issues that should be combined.

Harmful to the Population Health in this analysis refers to Fatalities and Injuries to persons. The property or economic damage will be in the second section of the analysis.

From names(raw_data) we know that we are interested in columns INJURIES and FATALITIES and the columns EVTYPE

We are safe to remove any rows that are zero for both INJURIES and FATALITIES as they will not help us in figuring out the Maximum Damage instead of subsetting one could assign them all to the no damage category

sum( ( pop.health$INJURIES == 0 ) & ( pop.health$FATALITIES == 0) )
## [1] 880368
pop.health.sm <- filter(pop.health, 
             !(( pop.health$INJURIES == 0 ) & ( pop.health$FATALITIES == 0)))

Still doesnt remove many of the EVTYPEs. Some of the EVTYPES are mangled versions of each other and should be combined. We can fix the lower case using toupper from dplyr. There are numerous spelling errors and versions of the same EVTYPE that need to be brute force changed to match.

library(reshape2)
dfn = as.data.frame(sapply(pop.health.sm, toupper))

## S3 method for class 'data.frame'
newdfn <- melt(dfn, id.vars=c("EVTYPE") )
## Warning: attributes are not identical across measure variables; they will
## be dropped
# Need to make them allcaps then more will match with group_by
# In dfn rows with matching EVTYPE and Variable
# should be summed and collapsed.
newdfn$value <- as.integer(newdfn$value)
df <- newdfn %>% group_by(EVTYPE, variable) %>% summarise_each(funs(sum))
df <- df[!(df$value == 0 ), ]
